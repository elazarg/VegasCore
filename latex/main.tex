\begin{abstract}
A finite multi-party game can be specified by a program: each move is a
guarded write, each random sample is a public coin, and payoffs are
deterministic functions of the terminal public state.  The strategically
relevant structure of such a program — who chooses what, what they can
observe, and which choices are independent — is determined not by the
syntax tree but by the dependency relation between events.  VegasCore
takes this observation as a design commitment: the carrier of
game-theoretic meaning is neither the source program nor a
hand-written extensive-form tree, but a finite, visibility-aware
\emph{event graph}.

Programs are checked for static visibility, freshness, reveal completeness,
distribution normalisation, and at-least-one-legal-action obligations,
then lowered definitionally to the event graph.  Configurations of the
graph are extensional partial assignments from event nodes to write
slices; the graph induces an asynchronous probabilistic
\emph{machine}.  Native round views of that machine define the
strategy carriers, game forms, and kernel games; round-batched
factored-observation stochastic-game (FOSG) and extensive-form-game
(EFG) presentations are derived as checked views.  This report develops the construction
mathematically, states its main contracts as theorems, and explains
the design choices that motivate them.

The headline results are: \textbf{frontier commutation} (independent
events commute extensionally), \textbf{commit/reveal information
barrier} (no reveal is enabled while a prior commit is still on the
frontier), \textbf{trace-to-public agreement} (the canonical pure and
PMF-behavioural kernel games agree with their blocked-trace
refinements after payoff projection, while terminal-public-state game
forms expose the non-payoff public observation separately),
\textbf{finite FOSG Kuhn realisation} with an explicit native/FOSG
strategy-equivalence obligation for the native kernel-game statement, and
\textbf{stochastic-refinement transport} (Nash equilibria pull back
along compatible backend refinements).  All theorems are mechanised in
Lean~4~\citep{deMouraUllrich2021} on top of mathlib~\citep{mathlib2020};
a companion definitional supplement records the exact Lean signatures
referenced here.
\end{abstract}
