\section{Introduction}
\label{sec:intro}

\subsection{Protocols vs.\ extensive forms}

Many games are easier to write down as protocols than as normal forms.
A player commits a value, a public random sample is drawn, a hidden
value is later revealed, and payoffs are computed from the public
transcript.  The mathematical game is implicit in these operational
steps: strategies are constrained by what players can observe at each
move, deviations must be legal at the point where they are taken, and
two presentations that linearise the same independent events in
different orders should denote the same game.

In the present construction, the primary semantic object is the
\emph{dependency relation between events} — a finite event structure
in the sense of \citet{NielsenPlotkinWinskel1981,Winskel1986EventStructures},
specialized to the protocol setting.  A program is checked, lowered to a finite directed acyclic
graph of events, and that graph is read as a probabilistic machine.
Strategic-form, FOSG, and EFG presentations of the machine are then
derived as checked \emph{views}, never as independent denotations.
The strategically irrelevant ordering of independent events is
quotiented out by construction: configurations of the graph are
extensional partial assignments from nodes to results, and the
frontier — the set of currently enabled events — is computed from
that assignment, not stored.

\subsection{A first example}
\label{sec:intro:matching-pennies}

Consider a Matching-Pennies-style protocol.  Two players, the
\textsf{matcher} and the \textsf{mismatcher}, each commit a private
boolean and later reveal it; the matcher is paid $+1$ on a match and
$-1$ otherwise, while the mismatcher's payoff is the matcher's
negated.  A natural protocol-style description is sequential:
\begin{quote}\small
\begin{tabular}{@{}l@{}}
$\Commit_{\textsf{m}}\,x_{\textsf{m}} \mid \mathtt{true};$\\
$\Commit_{\textsf{m'}}\,x_{\textsf{m'}} \mid \mathtt{true};$\\
$\Reveal\,y_{\textsf{m}}{:=}x_{\textsf{m}};$\\
$\Reveal\,y_{\textsf{m'}}{:=}x_{\textsf{m'}};$\\
$\Ret\,[\textsf{m}\!\mapsto\!\mathbb{1}_{y_{\textsf{m}}=y_{\textsf{m'}}} - \mathbb{1}_{y_{\textsf{m}}\neq y_{\textsf{m'}}},\
\textsf{m'}\!\mapsto\!-\,\mathbb{1}_{y_{\textsf{m}}=y_{\textsf{m'}}} + \mathbb{1}_{y_{\textsf{m}}\neq y_{\textsf{m'}}}].$
\end{tabular}
\end{quote}
The bindings $x_{\textsf{m}}, x_{\textsf{m'}}$ are hidden to their
respective owners; the bindings $y_{\textsf{m}}, y_{\textsf{m'}}$
are public.  Each $\Commit$ is annotated with a guard $\mathtt{true}$,
meaning the player may legally play any boolean.

This program is sequential, but the intended game is not: the two
commitments are simultaneous private moves.  As an extensive-form
tree, one must arbitrarily place either the matcher or the mismatcher
at the root, and use information-set equivalence to assert that
neither player observes the other's commit before making their own.

In the event-graph picture, the diamond

\node[ev] (cm) at (-2,1) {$\Commit_{\textsf{m}}\,x_{\textsf{m}}$};

\node[ev] (rm) at (-2,-1) {$\Reveal\,y_{\textsf{m}}$};

\end{tikzpicture}
\end{center}
displays exactly the strategic structure: the two commits share the
initial frontier, neither reveal can fire while a prior commit is
still pending (the cross-arrows from each commit into the
\emph{other} player's reveal are the protocol-level information
barrier), and the payoff node depends on both reveals.  We
distinguish two independent claims about this graph, both of which
the construction will mechanise:

\begin{description}[itemsep=2pt]
\item[(D) Diamond.]
  After both commits have completed, the configuration reached is
  independent of which commit was scheduled first.
\item[(B) Information barrier.]
  At any configuration where the matcher's reveal is on the
  frontier, the mismatcher's commit is no longer available — and
  symmetrically.  No player can condition a commit on observing
  another player's reveal.
\end{description}

(D) is structural: in the event-graph carrier, two configurations
that finish the same set of nodes with the same per-node results are
literally the same configuration, by extensionality.  (B) is a
theorem about the prerequisite relation generated by lowering: a
$\Reveal$ depends not only on its own committed payload but on every
lexically prior commit.  Both claims live in the
graph, not in the strategy carrier, which is why neither needs to be
restated for the FOSG, EFG, or kernel-game views derived later.

\subsection{The semantic spine}
\label{sec:intro:spine}

The construction is organized as a sequence of definitional arrows,
each owned by the source layer:
\begin{equation*}
  \mathsf{VegasLang}
  \;\xrightarrow{\;\mathsf{lower}\;}\;
  \mathsf{VegasCore}
  \;\xrightarrow{\;\mathsf{elaborate}\;}\;
  \mathsf{EventGraph}
  \;\xrightarrow{\;\mathsf{interpret}\;}\;
  \mathsf{Machine}.
\end{equation*}
$\mathsf{VegasLang}$ is the user-facing surface syntax (with
simultaneous yield phases, optional yields, and surface-level
ergonomics).  It lowers definitionally to the typed core
$\mathsf{VegasCore}$, which exposes five constructors:
$\Let$, $\Sample$, $\Commit$, $\Reveal$, $\Ret$.  A checked program
($\mathsf{WFProgram}$) bundles a core program with the static
obligations that turn it into a game description: contextual
freshness, reveal completeness, distribution normalization, view
scoping, and at-least-one-legal-action per commit site.  Elaboration
turns a checked program into an event graph; interpretation reads the
graph as a machine.

On top of the machine, three kinds of consumers are defined:

\begin{itemize}\itemsep1pt
\item \textbf{Strategic semantics} — strategy carriers, game forms,
  kernel games, expected utility, Nash equilibrium, dominance,
  correlated equilibria, potential predicates, and so on.  These
  Vegas-facing objects are built from the bounded native round view
  of the event-graph machine.  The FOSG view is a checked
  presentation used for FOSG/EFG/Kuhn bridge theorems, not the
  definition of the strategy carrier.
\item \textbf{Optional presentation bridges} — the canonical
  serialization of the FOSG view into an extensive-form tree.  The
  bridge is checked: the tree's payoff-vector outcome law agrees
  with the behavioral kernel game's.
\item \textbf{Backend refinement} — relations between an
  implementation machine (potentially with extra runtime structure,
  scheduling, internal events) and the canonical event-graph
  machine, with theorems transporting Nash equilibria from the
  specification to the implementation along compatible refinements.
\end{itemize}

\subsection{Contributions and theorem inventory}
\label{sec:intro:contributions}

This report is a description of the mechanized theory and the design
decisions behind it.  The theorem-level contributions are:

\begin{enumerate}[leftmargin=1.4em,itemsep=2pt]
\item \emph{Definitional lowering.}  $\mathsf{VegasLang}$ programs
  lower definitionally to $\mathsf{VegasCore}$; the nullable-yield
  compiler is provably equal to the canonical lowering pointwise.
  Concrete programs are checked by reflection.

\item \emph{Event-graph adequacy.}  The elaboration
  $\mathsf{VegasCore} \to \mathsf{EventGraph}$ produces a finite
  ranked DAG whose terminal-configuration outcome agrees with the
  declared payoff expressions.

\item \emph{Frontier diamond and preservation
  (\S\ref{sec:eventgraphs}).}  Distinct enabled nodes preserve each
  other's enablement and commute extensionally.

\item \emph{Commit/reveal information barrier
  (\S\ref{sec:eventgraphs}).}  No reveal is on the frontier of any
  configuration whose graph still has an unfinished prior commit.
  The strategy-surface corollary states that a player action for
  such a prior commit is unavailable while the reveal is on the
  frontier.

\item \emph{Trace-to-public agreement
  (\S\ref{sec:fosg-strategies}).}  The blocked-trace pure and
  PMF-behavioral kernel games agree with their payoff projections
  on outcome kernels and on expected utilities.  Terminal-public-
  state game forms expose the public observation separately from the
  payoff-vector kernel games.

\item \emph{Finite Kuhn realization
  (\S\ref{sec:presentations}).}  The bounded event-graph FOSG
  presentation satisfies the mixed-to-behavioral realization
  theorem.  The native kernel-game theorem is stated directly over
  $\mathrm{pureKernelGame}$ and $\mathrm{pmfBehavioralKernelGame}$,
  using the proved native/FOSG outcome-law transport.

\item \emph{EFG presentation
  (\S\ref{sec:presentations}).}  The canonical FOSG-to-EFG bridge
  preserves payoff-vector outcome laws.

\item \emph{Stochastic-refinement transport
  (\S\ref{sec:refinement}).}  A stochastic step refinement plus a
  compatible block-law lift yields an expected-utility-preserving
  morphism from a backend kernel game into the canonical
  blocked-trace kernel game; Nash equilibria pull back along this
  morphism.
\end{enumerate}
