\section{FOSG, EFG, and Kuhn presentations}
\label{sec:presentations}

\subsection{FOSG as a presentation}

FOSG is the standard imperfect-information game-form presentation used
by the broader library.  VegasCore does not define its strategic
semantics by first translating to FOSG; native frontier games own the
Vegas-facing semantics.  Instead, the checked frontier round view is
exported as a bounded FOSG together with payoff-faithfulness theorems.

The FOSG bridge is still mathematically central.  It states that the
payoff-facing FOSG history kernel has the same joint utility
distribution as the native behavioral frontier game, and that every
exported FOSG profile restricts back to a native behavioral profile with
the same utility distribution.  Thus FOSG is the standard presentation
through which the paper can connect VegasCore to existing
imperfect-information game theory without making FOSG an implementation
IR.

\subsection{EFG presentation}
\label{sec:pres:efg}

The EFG layer serializes the bounded FOSG presentation through the
generic FOSG-to-Augmented-EFG bridge.  The resulting tree is a
presentation of the same payoff law.  It necessarily introduces a
sequential representation of simultaneous or checkpointed actions, so it
is not treated as the semantic owner.

\begin{theorem}[EFG payoff adequacy]
\label{thm:efg}
For every checked program and every native behavioral frontier profile,
translation through the FOSG/EFG bridge preserves the joint utility
distribution of the native behavioral frontier game.
\end{theorem}

\subsection{Kuhn at the frontier information model}
\label{sec:pres:kuhn}

Kuhn realization is stated for the canonical frontier information model:
players reason about reachable checkpoint observations and available
menus induced by the checked program.  The main theorem has two
directions.

\begin{theorem}[Behavioral to product mixed pure]
\label{thm:kuhn-beh-to-mixed}
Every behavioral frontier profile has a product mixed-pure frontier
profile with the same payoff-vector outcome law.
\end{theorem}

\begin{theorem}[Product mixed pure to behavioral]
\label{thm:kuhn-mixed-to-beh}
Every product mixed-pure frontier profile has a behavioral frontier
profile with the same payoff-vector outcome law.  Moreover the
translation is Nash-safe: unilateral deviations on the target side have
matching unilateral deviations on the source side with the same relevant
outcome law.
\end{theorem}

These are not merely EFG export facts.  They are program-facing facts
about the frontier games produced from VegasCore.  The public
checked-program theorem is packaged by the native completed-frontier
RoundView/Kuhn development: \(\mathrm{frontierCompleteOutcomeKuhn}\)
relates \(\mathrm{behavioralFrontierGame}\),
\(\mathrm{mixedPureFrontierGame}\), and
\(\mathrm{pureFrontierGame}\) at outcome-kernel level, and the
deviation-simulation wrappers provide Nash-safe transport.  FOSG and
EFG adequacy theorems present the same frontier semantics to external
game forms; they are not the semantic reduction behind this Kuhn
statement.
