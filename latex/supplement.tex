\begin{abstract}
This supplement is the code-facing companion to the VegasCore report.
It records the current Lean architecture and the main definition and
theorem names used by the paper.  It is intentionally a supplement to
VegasCore itself: the generic GameTheory library is a dependency and a
target of bridge theorems, not a place where VegasCore-specific
semantics or documentation live.
\end{abstract}

\section{Pipeline}

The maintained semantic spine is:
\[
  \TT{VegasLang}
  \to
  \TT{VegasCore}
  \to
  \TT{EventGraph.Graph}
  \to
  \TT{Machine}
  \to
  \TT{KernelGame}.
\]
The first arrow is surface lowering.  It erases administrative
\(\Let\)-bindings and lowers nullable yields to hidden optional
commitments followed by reveals.  The second arrow is graph
compilation.  The third is the primitive operational adapter.  The
fourth is the frontier round-view construction that gives native
strategic games.

\section{Source Syntax}

\subsection{Core}

\begin{leanbox}
inductive VegasCore (Player : Type) [DecidableEq Player] (L : IExpr) :
    VCtx Player L -> Type
| ret    : List (Player \(\times\) L.Expr (erasePubVCtx Gamma) L.int) -> ...
| sample : VarId -> L.DistExpr (erasePubVCtx Gamma) b -> ...
| commit : VarId -> Player ->
    L.Expr ((x, b) :: eraseVCtx (viewVCtx who Gamma)) L.bool -> ...
| reveal : VarId -> Player -> VarId ->
    VHasVar Gamma x (.hidden who b) -> ...
\end{leanbox}

\TT{VegasCore} is the maintained protocol core.  It has no
administrative let constructor.  Samples are public.  Commits bind
hidden values owned by the committing player.  Reveals introduce fresh
public aliases for hidden values.  Terminal payoff expressions read the
public context.

\subsection{Surface}

\begin{leanbox}
inductive VegasLang (P : Type) [DecidableEq P] :
    VCtx P simpleExpr -> Type
| ret
| letExpr
| sample
| commit
| yield
| reveal

def VegasLang.lower : VegasLang P Gamma -> VegasCore P simpleExpr Gamma
\end{leanbox}

\TT{VegasLang} is a concrete surface layer.  Its \TT{letExpr}
constructor is erased by the lowering environment.  Its \TT{yield}
constructor lowers to a nullable hidden commitment plus public reveal.
This is why deterministic administrative bindings do not allocate graph
events.

\subsection{Checked-program bundles}

\begin{leanbox}
structure GraphProgram (P : Type) [DecidableEq P] (L : IExpr) where
  Gamma      : VCtx P L
  prog       : VegasCore P L Gamma
  env        : VEnv L Gamma
  wctx       : WFCtx Gamma
  fresh      : FreshBindings prog
  normalized : NormalizedDists prog

structure WFProgram (P : Type) [DecidableEq P] (L : IExpr) where
  core    : GraphProgram P L
  reveals : RevealComplete [] core.prog
  legal   : Legal core.prog
\end{leanbox}

\TT{GraphProgram} is the minimal graph-construction input.
\TT{WFProgram} is the checked game boundary.  The compiler consumes the
former; progress and game-facing theorems consume the latter.

\section{Event Graphs}

\subsection{Graph rows}

\begin{leanbox}
structure InitialField (Player : Type) (L : IExpr) where
  ty    : L.Ty
  owner : Option Player
  value : L.Val ty

structure EventNode (Player : Type) (L : IExpr) where
  ty    : L.Ty
  owner : Option Player
  sem   : NodeSem Player L

structure Graph (Player : Type) [DecidableEq Player] (L : IExpr) where
  initialFields : List (InitialField Player L)
  nodes         : List (EventNode Player L)
\end{leanbox}

Initial fields occupy the first field ids.  Event output fields are
derived as \TT{initialFields.length + node}.  There is no stored
writer map and no stored prerequisite map.

\subsection{Read environments and node semantics}

\begin{leanbox}
structure FieldRef (L : IExpr) where
  field : Nat
  ty    : L.Ty

structure ReadEnv (L : IExpr) (reads : Finset (FieldRef L)) where
  read : (ref : FieldRef L) -> ref \(\in\) reads -> L.Val ref.ty

inductive NodeSem (Player : Type) (L : IExpr)
| sample (dist : EventDist L)
| commit (who : Player) (guard : EventGuard L)
| reveal (source : Nat)
\end{leanbox}

Graph-local computations receive only a \TT{ReadEnv} for their declared
typed footprint.  This makes read annotations the semantic boundary for
scheduling and observation.

\subsection{Well-formedness and prerequisites}

\begin{leanbox}
def Graph.nodeWFAt (G : Graph Player L) (node : Nat)
    (event : EventNode Player L) : Prop := ...

def Graph.WF (G : Graph Player L) : Prop :=
  forall node event, G.nodes[node]? = some event -> G.nodeWFAt node event

def Graph.prereqs (G : Graph Player L)
    (node : Fin G.nodeCount) : Finset (Fin G.nodeCount) := ...
\end{leanbox}

\TT{Graph.WF} enforces id validity, type compatibility, read
availability, public reads for internal events with public output, actor
visibility for commit choice footprints, and hidden-to-public discipline for
reveals.  \TT{Graph.prereqs} is canonical: a prior node is a
prerequisite exactly when it writes a field read by the current node,
or when the current node is a reveal and the prior node is a commit.

\subsection{Execution}

\begin{leanbox}
structure Config (G : Graph Player L) where
  done  : Finset (Fin G.nodeCount)
  store : Store L

def Ready (G : Graph Player L) (cfg : Config G)
    (node : Fin G.nodeCount) : Prop := ...

  node  : Fin G.nodeCount

Raw configurations are low-level data.  The primitive machine uses
\TT{ReachableConfig}, so terminal payoff totality is proved for
reachable terminal states rather than assumed for arbitrary raw states.

\section{Compiler}

\begin{leanbox}
structure CompiledProgram (P : Type) [DecidableEq P] (L : IExpr) where
  graph     : EventGraph.Graph P L
  graphWF   : graph.WF
  payoffs   : List (P \(\times\) EventPayoff L)
  payoffsWF :
    forall payoff, payoff \(\in\) payoffs ->
      forall ref, ref \(\in\) payoff.2.reads ->
        graph.fieldRefPublic ref /\
        graph.fieldAvailableBefore graph.nodeCount ref.field = true

noncomputable def compile (g : GraphProgram P L) : CompiledProgram P L
\end{leanbox}

The compiler is indexed by typed source contexts.  Variable lookup is
proof-bearing; missing variables do not default to a field id.  Public
surface lets have already been substituted away.  Commit reads are
computed from the committing player's visible current fields; sample and
payoff reads use declared public dependencies; reveals read the hidden
source field.

Important theorem families:
\begin{itemize}\itemsep1pt
\item \TT{compile_guardLive}: source \TT{Legal} implies graph
  \TT{GuardLive};
\item \TT{compiledGraph_wf}: checked programs compile to graph-WF data;
\item \TT{primitiveOutcome_eq_sourceAtTerminal}: primitive terminal
  outcomes agree with source payoff reconstruction;
\item \TT{pureFrontierOutcomeKernel_sourceMap}: pure frontier kernels
  are source-payoff projections of canonical run distributions;
\item \TT{behavioralFrontierOutcomeKernel_sourceMap}: behavioral
  frontier kernels satisfy the analogous projection law.
\end{itemize}

\section{Primitive Machines and Checkpoints}

\begin{leanbox}
structure GraphMachineSpec (G : Graph P L) where
  graphWF  : G.WF
  payoffs  : List (P \(\times\) EventPayoff L)
  payoffsWF : ...

noncomputable def EventGraph.ToMachine.primitiveMachine
    (spec : GraphMachineSpec G) : Machine P
\end{leanbox}

The primitive machine exposes one available graph event per primitive
step.  Total machine transitions stutter on unavailable raw events, but
legal event laws and frontier presentations use availability evidence.

\begin{leanbox}
inductive BatchStep (G : Graph Player L) :
    ReachableConfig G -> ReachableConfig G -> Type

structure CheckpointPresentation (G : Graph Player L) where ...
\end{leanbox}

Batches are ordered operational witnesses.  Checkpoint presentations
expose allowed source/destination checkpoints and hide the primitive
order from strategic histories.

\TT{FrontierRoundSemantics} certifies support: every positive-mass
checkpoint successor has a replayable primitive explanation.  Primitive
event-batch probability laws are handled by the checked-program trace and
runtime-refinement adequacy layers, where the recorded event batches are
part of the statement.

\section{Frontier Games}

\begin{leanbox}
noncomputable def frontierPresentation
noncomputable def canonicalFrontierRoundSemantics
noncomputable def frontierSemantics (program : WFProgram P L)

noncomputable def pureFrontierGame
noncomputable def behavioralFrontierGame
noncomputable def mixedPureFrontierGame
\end{leanbox}

The frontier round view first closes ready internal work.  If no
internal work is ready, active players choose for ready commit nodes,
then the transition closes newly ready internal work.  The finite-domain
evidence supplies finite action and state carriers.

The three canonical kernel games are the Vegas-facing strategic
denotations:
\begin{itemize}\itemsep1pt
\item pure reachable frontier strategies;
\item behavioral reachable frontier strategies;
\item product mixed pure strategies.
\end{itemize}

\section{FOSG, EFG, and Kuhn Interfaces}

\begin{leanbox}
noncomputable def frontierFOSG
noncomputable def frontierPlainEFG
\end{leanbox}

FOSG and EFG are presentation/export layers.  Their theorem surface
states utility-distribution agreement with native frontier games, not a
replacement definition of Vegas semantics.

\begin{leanbox}[fontupper=\scriptsize\ttfamily]
theorem mixedPureFrontier\_udist\_of\_behavioral\\
theorem behavioralFrontier\_udist\_of\_mixedPure\\
theorem frontierCompleteOutcomeKuhn\_schema\\
noncomputable def frontierKuhnStrategicEquivalence\_schema\\
theorem mixedPureFrontier\_mixedDeviation\_to\_behavioral\_kernel\\
theorem mixedPureFrontier\_behavioralDeviation\_to\_mixedPure\_kernel\\
theorem behavioralFrontier\_deviation\_to\_mixedPure\_kernel
\end{leanbox}

The public checked-program Kuhn path is native completed-frontier
packaging.  \TT{frontierCompleteOutcomeKuhn} packages the two
outcome-kernel directions over the completed frontier games.
\TT{frontierKuhnStrategicEquivalence\_schema} packages the stronger
strategy/deviation-level statement: canonical profile translations
preserve completed outcome kernels, and one-player deviations can be
matched across the mixed-pure and behavioral presentations.  Nash-safe
transport is a downstream use of these deviation facts.  FOSG and EFG
exports are payoff-law presentations that can be composed with these
facts; they are not the proof route for the public frontier Kuhn theorem.

\section{Runtime Refinement}

\begin{leanbox}
structure TraceGameSurface
    (program : WFProgram Player L) [FiniteDomains program] where
  game : GameTheory.KernelGame Player
  traceKernel :
    (forall player, game.Strategy player) ->
      PMF ((ToEventGraph.PrimitiveMachine
        (ToEventGraph.compile program.core)).EventBatchTrace)
  traceKernel\_udist :
    forall profile,
      (traceKernel profile).bind
        (fun trace => PMF.pure
          (Machine.eventBatchTraceUtility
            (ToEventGraph.PrimitiveMachine
              (ToEventGraph.compile program.core))
            (fun any => 0) trace)) =
      game.udist profile

structure TraceSpecEventBatchLaw
    (program : WFProgram Player L) [FiniteDomains program]
    (surface : TraceGameSurface program) where
  specLaw :
    (forall player, surface.game.Strategy player) ->
      (ToEventGraph.PrimitiveMachine
        (ToEventGraph.compile program.core)).EventBatchLaw
  legal :
    forall profile,
      (ToEventGraph.PrimitiveMachine
        (ToEventGraph.compile program.core)).IsLegalEventBatchLaw
        (specLaw profile)
  trace\_eq :
    forall profile,
      (ToEventGraph.PrimitiveMachine
        (ToEventGraph.compile program.core)).eventBatchTraceDist
        (specLaw profile) program.frontierSemantics.horizon =
      surface.traceKernel profile

structure RuntimeTraceAdequacy
    (program : WFProgram Player L) [FiniteDomains program]
    (surface : TraceGameSurface program)
    {Impl : Machine Player}
    (R : Machine.StochasticRefinement Impl
      (ToEventGraph.PrimitiveMachine
        (ToEventGraph.compile program.core))) where
  spec : TraceSpecEventBatchLaw program surface
  impl : Impl.EventBatchLawFamily surface.game.Strategy
  compatible :
    forall profile,
      R.EventBatchLawCompatible
        (impl.law profile) (spec.specLaw profile)
\end{leanbox}

Canonical surfaces are:
\begin{itemize}\itemsep1pt
\item \TT{behavioralFrontierTraceSurface};
\item \TT{pureFrontierTraceSurface};
\item \TT{mixedPureFrontierTraceSurface}.
\end{itemize}
The main transport theorem is
\TT{RuntimeTraceAdequacy.implTraceGame_udist_surface}.  Under
bounded-utility hypotheses, \TT{implTraceGame_nash_of_surface_nash}
pulls surface Nash profiles back to the implementation trace game.

\section{Paper-facing Theorem Surface}

\TT{Vegas.Core.Theorems.Claims} collects theorem statements whose names
mirror the paper claims.  The module covers compilation to well-formed
source-free event graphs, checkpoint progress, terminal source-payoff
adequacy, native frontier payoff adequacy, FOSG and EFG presentation
adequacy, native frontier Kuhn realization, round-view locality of
behavioral strategies, and runtime-refinement transport for utility
distributions and Nash profiles.  The implementation-facing theorem map
is maintained in \TT{docs/semantics-spine.md}; this supplement records
the semantic categories rather than a second exhaustive theorem list.

\section{Examples and Tests}

End-to-end examples exercise language lowering, event-graph
dependencies, frontier games, solution concepts, FOSG/EFG bridges, and
refinement smoke tests.  The build target for these fixtures is separate
from the top-level library export so that examples can remain regression
tests without becoming the public import surface.
